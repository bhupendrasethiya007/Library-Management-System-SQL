# Library Management System using SQL

## Project Overview

This project implements a Library Management System using PostgreSQL. The database is designed to manage branches, employees, members, books, book issuance, and book returns. The project covers database design, CRUD operations, business reporting, and advanced SQL queries to analyze library operations and member activity.

---

## Database Schema

The system consists of six relational tables:

### Branch

Stores branch information.

| Column         | Description       |
| -------------- | ----------------- |
| branch_id      | Branch Identifier |
| manager_id     | Branch Manager ID |
| branch_address | Branch Address    |
| contact_no     | Contact Number    |

### Employees

Stores employee details.

| Column    | Description       |
| --------- | ----------------- |
| emp_id    | Employee ID       |
| emp_name  | Employee Name     |
| position  | Job Position      |
| salary    | Employee Salary   |
| branch_id | Associated Branch |

### Members

Stores library member information.

| Column         | Description       |
| -------------- | ----------------- |
| member_id      | Member ID         |
| member_name    | Member Name       |
| member_address | Address           |
| reg_date       | Registration Date |

### Books

Stores book inventory.

| Column       | Description         |
| ------------ | ------------------- |
| isbn         | Book ISBN           |
| book_title   | Book Title          |
| category     | Book Category       |
| rental_price | Rental Price        |
| status       | Availability Status |
| author       | Author Name         |
| publisher    | Publisher           |

### Issued Status

Tracks issued books.

| Column           | Description |
| ---------------- | ----------- |
| issued_id        | Issue ID    |
| issued_member_id | Member ID   |
| issued_book_name | Book Name   |
| issued_date      | Issue Date  |
| issued_book_isbn | ISBN        |
| issued_emp_id    | Employee ID |

### Return Status

Tracks returned books.

| Column           | Description      |
| ---------------- | ---------------- |
| return_id        | Return ID        |
| issued_id        | Related Issue ID |
| return_book_name | Book Name        |
| return_date      | Return Date      |
| return_book_isbn | ISBN             |

## Task 5: List Members Who Have Issued More Than One Book

**Question:** Identify members who have issued multiple books.

```sql

       COUNT(*) AS total_books_issued
FROM issued_status5
```

## Task 6: Create Book Issue Summary

**Question:** Find the total number of times each book has been issued.

```sql
SELECT b.book_title,
       b.isbn,
       COUNT(ist.issued_book_name) AS total_issued
FROM books4 b
JOIN issued_status5 ist
ON b.isbn = ist.issued_book_isbn
GROUP BY b.book_title, b.isbn;
```

## Task 8: Find Total Rental Income by Category

**Question:** Calculate rental revenue generated by each category.

```sql

       SUM(b.rental_price) AS total_revenue,
       COUNT(ist.issued_id) AS total_issues
FROM books4 b
JOIN issued_status5 ist
ON b.isbn = ist.issued_book_isbn
```

## Task 13: Identify Members with Overdue Books

**Question:** Find members who have not returned books within 30 days.

```sql
SELECT s.issued_member_id,
       m.member_name,
       b.book_title,
       s.issued_date,
       CURRENT_DATE - s.issued_date AS overdue_days
FROM members3 m
JOIN issued_status5 s
ON m.member_id = s.issued_member_id
JOIN books4 b
ON b.isbn = s.issued_book_isbn
LEFT JOIN return_status6 r
ON r.issued_id = s.issued_id
WHERE r.return_date IS NULL
AND (CURRENT_DATE - s.issued_date) > 30;
```

## Task 15: Branch Performance Report

**Question:** Generate a branch-wise performance report.

```sql
SELECT br.branch_id,
       br.manager_id,
       COUNT(s.issued_id) AS total_books_issued,
       COUNT(r.return_id) AS total_books_returned,
       SUM(b.rental_price) AS total_revenue
FROM issued_status5 s
JOIN employees2 e
ON e.emp_id = s.issued_emp_id
JOIN branch1 br
ON e.branch_id = br.branch_id
LEFT JOIN return_status6 r
ON r.issued_id = s.issued_id
JOIN books4 b
ON s.issued_book_isbn = b.isbn
GROUP BY br.branch_id, br.manager_id;
```

## Task 16: Create Active Members Table (CTAS)

**Question:** Create a table containing members who issued books in the last month.

```sql
CREATE TABLE active_members AS
SELECT *
FROM members3
WHERE member_id IN (
    SELECT issued_member_id
    FROM issued_status5
    WHERE issued_date >= CURRENT_DATE - INTERVAL '1 month'
);
```

---

## Task 17: Find Employees with the Most Book Issues Processed

**Question:** Find the top 3 employees who processed the highest number of book issues.

```sql
SELECT e.emp_name,
       b.branch_id,
       COUNT(s.issued_id) AS total_books_issued
FROM employees2 e
JOIN issued_status5 s
ON e.emp_id = s.issued_emp_id
JOIN branch1 b
ON e.branch_id = b.branch_id
GROUP BY e.emp_name, b.branch_id
ORDER BY total_books_issued DESC
LIMIT 3;
```

---

## Task 18: Identify High-Risk Members

**Question:** Find members who have issued the same book more than twice.

```sql
SELECT m.member_name,
       b.book_title,
       COUNT(*) AS total_issues,
       CASE
           WHEN COUNT(*) > 2 THEN 'High Risk'
           ELSE 'Normal'
       END AS status
FROM books4 b
JOIN issued_status5 s
ON b.isbn = s.issued_book_isbn
JOIN members3 m
ON m.member_id = s.issued_member_id
GROUP BY m.member_name, b.book_title
HAVING COUNT(*) > 2;
```
